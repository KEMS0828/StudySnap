# グループ管理・投稿編集・退会処理の4つの修正

## 1. 退会済みユーザーの投稿をそのまま残す
- アカウント削除時、投稿・写真・セッション・チャットメッセージは削除せず、そのまま残るように変更
- ユーザー名もそのまま表示される（「退会したユーザー」などに置き換えない）
- プロフィール情報（ユーザー自身のデータ）と目標・通知のみ削除

## 2. グループ名・詳細を管理者が編集できる
- 設定 → グループ管理画面の上部に「グループ情報を編集」ボタンを追加
- 編集画面ではグループ画像・グループ名・説明・参加方法（自由参加 / 承認制）を変更可能
- 保存するとグループ詳細・一覧にすぐ反映される
- 管理者のみアクセス可能

## 3. 投稿編集画面でキーボードが閉じれるように
- 投稿編集画面のキーボード上部に「完了」ボタンを追加してワンタップで閉じれるように
- 画面タップでもキーボードが閉じるように挙動を改善

## 4. 管理者によるメンバー退会を実際に反映
- 管理者が退会させたメンバーは、サーバー上でも所属グループからきちんと外される（現在も実装済みだが、退会されたユーザーのアプリ側で検知されていない問題を修正）
- 退会させられたユーザーが次にアプリを起動 or タイムラインを更新した時に、自分が所属グループから外れていることを検知
- 検知すると自動的にローカル状態をリセットし、グループ検索画面に戻す
- 「グループから退会させられました」という案内メッセージを1回だけ表示
