# Xキャンペーン用プロモコード（生涯無制限）機能を追加

## やりたいこと

Xの広告で「先着◯名様に生涯無制限プランをプレゼント！」という訴求をするため、ユーザーが特別なプロモコード（例: `XPROMO2026`）を入力すると、生涯無料で Pro 機能が使えるようにします。

## 機能

### ユーザー側

- ペイウォール画面の下部に「コードをお持ちの方はこちら」というリンクを追加
- タップすると、プロモコード入力シートが開く
- コードを入力して「適用」を押すと、即座に生涯 Pro 権利が付与される
- 既に使用済み・無効・先着枠終了の場合は分かりやすいエラー表示
- 一度コードを使えば、以降そのアカウントでは永久に Pro 扱い（ログインしているデバイス全てで同期）

### あなた（運営）側の運用

- Firestore に `promoCodes` コレクションを作るだけでコード発行可能（管理画面不要）
- 各コードドキュメントには次のフィールドを保存：
  - `code`（例: `XPROMO2026`）
  - `maxRedemptions`（先着人数、例: 100）
  - `currentRedemptions`（自動カウント）
  - `expiresAt`（任意の有効期限）
  - `isActive`（true/false で即時停止可能）
- コードを使ったユーザーは `redemptions` コレクションに記録され、二重利用を防止
- ユーザーの `users` ドキュメントに `lifetimePremium: true` フラグが立ち、アプリ側はこれを見て Pro 扱い

### 整合性

- Firestore のトランザクション処理で「先着 N 人」を厳密にカウント（同時アクセスでも超過しない）
- 同じユーザーが同じコードを 2 回使えないようにチェック
- アプリ起動時に Firestore の `lifetimePremium` を読み込み、RevenueCat のサブスクと合わせて Pro 判定

## デザイン

- ペイウォールの「購入を復元」ボタンの近くに、控えめな小さい文字で「🎁 プロモコードをお持ちの方」リンク
- コード入力シートはミニマルな白背景、中央にコード入力欄（大きめ・等幅フォント）、下にオレンジの「適用する」ボタン
- 適用成功時はチェックマークアニメーション + 「生涯 Pro が有効になりました 🎉」のトーストを表示してシートが閉じる
- エラー時は赤字でメッセージ（「このコードは見つかりません」「先着枠が終了しました」「すでに使用済みです」など）

## 画面

- **ペイウォール**: 既存の画面に「プロモコードをお持ちの方」リンクを追加
- **プロモコード入力シート（新規）**: コードを入力して適用するだけのシンプルなシート

## 注意点

- Firestore の `promoCodes` / `redemptions` コレクションには適切なセキュリティルールが必要です（コードドキュメントの read は許可、write は禁止；redemption 作成は本人のみ）。ルールはこの実装と合わせてご案内します。
- App Store の審査で問題にならないよう、これは「販促コード（マーケティングキャンペーン）」であり、購入の代替ではない位置づけにします（リンク文言を「キャンペーンコード」にする等）。
